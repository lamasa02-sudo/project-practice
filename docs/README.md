Целью отчёта является систематизация и описание выполненных работ, анализ достигнутых результатов и оценка их практической ценности для организации-заказчика — ООО «ВК». ## 1. Общая информация о проекте

### 1.1. Название проекта
Проект носит название **«Разработка веб-сайта для игры Dark Machine»**.  
Dark Machine — это 2D сюжетная метроидвания с реализованным core-геймплеем, базовой системой выбора, проработанными локациями и атмосферным визуальным оформлением, создаваемая командой независимых разработчиков.   Данный сайт разрабатывается в качестве практической работы в рамках проектной практики студента первого курса, что позволяет применить полученные теоретические знания по веб-разработке к реальной задаче, востребованной индустриальным партнёром.

### 1.2. Цели и задачи проекта
Основной целью проекта являлось создание информативного, визуально атмосферного и технически грамотного веб-ресурса, который представит игру Dark Machine целевой аудитории и потенциальным издателям, а также разработка сопутствующего инструментария для удобной работы с игровой документацией.

Для достижения цели решены следующие задачи:
1. Изучить концепцию игры, её ключевые механики (core-геймплей, система выбора) и особенности художественного стиля, чтобы отразить их в структуре и оформлении сайта.
2. Спроектировать и вручную сверстать статический веб-сайт, включающий аннотацию проекта, описание геймплея и локаций, информацию о команде разработчиков, журнал новостей и подборку полезных ресурсов по геймдизайну.
3. Освоить систему контроля версий Git и язык разметки Markdown для ведения технической документации, журнала прогресса и организации репозитория.
4. В рамках вариативной части задания «Практическая реализация технологии» разработать нативный текстовый редактор, оптимизированный для написания и рецензирования геймдизайн-документов, сценариев и внутриигровых текстов.
5. Визуализировать архитектуру проекта и структуру сайта с помощью диаграмм, включив их в техническое руководство.
6. Подготовить итоговую отчётную документацию с описанием всех этапов и результатов.

### 4.a. Веб-сайт (HTML/CSS)
Создан статический веб-сайт, полностью написанный на HTML5 и CSS3. Сайт развёрнут с помощью GitHub Pages и доступен по ссылке: https://lamasa02-sudo.github.io/project-practice/

Структура сайта логически отражает ключевые аспекты игры:
- Домашняя страница (index.html) — встречает посетителя атмосферным скриншотом из игры, содержит краткую аннотацию проекта и навигационное меню, стилизованное под внутриигровой интерфейс. (Приложение Б)
- Страница «О проекте» (about.html) — детально описывает концепцию 2D сюжетной метроидвании: реализованный core-геймплей, базовую систему выбора с ветвлением, проработанные локации и особенности атмосферного оформления. (Приложение В)
- Страница «Команда» (team.html) — представляет ключевых участников студии и личный вклад практиканта в создание сайта и инструментария. (Приложение Г)
- Страница «Журнал разработки» (journal.html) — содержит три новостных поста. (Приложение Д)
- Страница «Ресурсы» (resources.html) — включает ссылки на руководства по HTML и CSS, статьи по геймдизайну метроидваний и официальные ресурсы студии. (Приложение Е)

При вёрстке использовалась технология Flexbox для адаптивного отображения на различных устройствах. Цветовая гамма сайта выдержана в тёмных, индустриальных тонах, соответствующих визуальному стилю игры. Навигационные элементы снабжены CSS-эффектами при наведении, имитирующими подсветку элементов внутриигрового HUD.

### 4.b. Документация в Markdown и Git
Система контроля версий Git применялась на протяжении всего периода практики. Репозиторий на GitHub организован по модели с двумя основными ветками: main (стабильные релизы) и develop (рабочая версия). Все значимые изменения зафиксированы в виде атомарных коммитов с информативными сообщениями.

Документация выполнена в Markdown и включает:
1. README.md — введение в проект, инструкции по локальному запуску сайта и редактора.
2. docs/progress_journal.md — журнал прогресса.
3. docs/editor_tech_guide.md — техническое руководство по созданию редактора для геймдизайн-документов, содержащее примеры кода и описание архитектуры.
4. reports/report.docx и reports/report.pdf — итоговые версии отчёта.

### 4.c. Редактор текста
В рамках вариативной части практики разработан нативный текстовый редактор «GDD Editor» (Game Design Document Editor). Выбор технологического стека (Python 3.11, Tkinter) обеспечил простоту развёртывания и кроссплатформенность.

Выполнена творческая модификация проекта: вместо стандартного меню Tkinter разработана панель быстрого доступа, визуально перекликающаяся с элементами интерфейса сайта Dark Machine. (Приложение Ж)

def apply_dark_machine_style():
    """Применяет стиль Dark Machine ко всему приложению"""
    style = ttk.Style()
    style.theme_use("clam")
    style.configure("Dark.TFrame", background=DARK_BG)
    style.configure("Dark.TLabel", background=DARK_BG, foreground=ACCENT_RED_LIGHT, font=("Consolas", 9))
    style.configure("Dark.TButton", background=PANEL_BG, foreground=TEXT_COLOR, borderwidth=1, focusthickness=0, font=("Consolas", 9))
    style.map("Dark.TButton", background=[("active", ACCENT_RED), ("pressed", ACCENT_RED_LIGHT)], foreground=[("active", "white")])

def create_mechanical_button(parent, text, command, row=None, col=None, padx=4, pady=4):
    """Создаёт кнопку в стиле механических тумблеров"""
    btn = tk.Button(
        parent,
        text=text,
        command=command,
        bg=PANEL_BG,
        fg=TEXT_COLOR,
        activebackground=ACCENT_RED,
        activeforeground="white",
        relief="flat",
        bd=1,
        padx=12,
        pady=6,
        font=("Consolas", 9, "bold"),
        cursor="hand2"
    )
    
    def on_enter(e):
        btn.config(bg=ACCENT_RED, fg="white")
    
    def on_leave(e):
        btn.config(bg=PANEL_BG, fg=TEXT_COLOR)
    
    btn.bind("<Enter>", on_enter)
    btn.bind("<Leave>", on_leave)
    
    if row is not None and col is not None:
        btn.grid(row=row, column=col, padx=padx, pady=pady, sticky="ew")
    else:
        btn.pack(side="left", padx=padx, pady=pady)
    
    return btn

# ========== ОБНОВЛЕНИЕ СОСТОЯНИЯ ==========
def update_status(event=None):
    """Обновляет счётчик символов и статистику"""
    text = text_area.get("1.0", tk.END)
    chars = len(text) - 1
    words = len(text.split())
    lines = len(text.splitlines())
    status_text = f" СИМВОЛЫ: {chars} | СЛОВА: {words} | СТРОКИ: {lines}"
    status_bar.config(text=status_text)
    return chars

def open_file():
    file_path = filedialog.askopenfilename(
        filetypes=[("Dark Machine Script", "*.dms"), ("Text files", "*.txt"), ("All files", "*.*")]
    )
    if file_path:
        with open(file_path, "r", encoding="utf-8") as file:
            text_area.delete("1.0", tk.END)
            text_area.insert(tk.END, file.read())
        update_status()
        root.title(f"DARK MACHINE — {os.path.basename(file_path)}")

def save_file():
    file_path = filedialog.asksaveasfilename(
        defaultextension=".dms",
        filetypes=[("Dark Machine Script", "*.dms"), ("Text files", "*.txt"), ("All files", "*.*")]
    )
    if file_path:
        with open(file_path, "w", encoding="utf-8") as file:
            file.write(text_area.get("1.0", tk.END))
        update_status()
        root.title(f"DARK MACHINE — {os.path.basename(file_path)}")

def new_file():
    if text_area.get("1.0", tk.END).strip():
        if messagebox.askyesno("DARK MACHINE", "Сохранить текущий сценарий перед созданием нового?"):
            save_file()
    text_area.delete("1.0", tk.END)
    root.title("DARK MACHINE — НОВЫЙ СЦЕНАРИЙ")
    update_status()

def export_to_json():
    """Экспорт сценария в JSON для движка игры"""
    file_path = filedialog.asksaveasfilename(
        defaultextension=".json",
        filetypes=[("JSON files", "*.json")]
    )
    if file_path:
        script_data = {
            "title": root.title(),
            "content": text_area.get("1.0", tk.END),
            "export_date": datetime.now().isoformat(),
            "word_count": len(text_area.get("1.0", tk.END).split()),
            "char_count": update_status()
        }
        with open(file_path, "w", encoding="utf-8") as file:
            json.dump(script_data, file, indent=2, ensure_ascii=False)
        messagebox.showinfo("DARK MACHINE", "Сценарий экспортирован в JSON")

def find_replace():
    win = tk.Toplevel(root)
    win.title("DARK MACHINE — ПОИСК И ЗАМЕНА")
    win.geometry("400x250")
    win.configure(bg=DARK_BG)
    tk.Label(win, text="ПОИСК:", bg=DARK_BG, fg=ACCENT_RED_LIGHT, font=("Consolas", 10, "bold")).pack(pady=10)
    find_entry = tk.Entry(win, width=40, bg=PANEL_BG, fg=TEXT_COLOR, insertbackground=ACCENT_RED_LIGHT, font=("Consolas", 10))
    find_entry.pack(pady=5)
    tk.Label(win, text="ЗАМЕНИТЬ НА:", bg=DARK_BG, fg=ACCENT_RED_LIGHT, font=("Consolas", 10, "bold")).pack(pady=10)
    replace_entry = tk.Entry(win, width=40, bg=PANEL_BG, fg=TEXT_COLOR, insertbackground=ACCENT_RED_LIGHT, font=("Consolas", 10))
    replace_entry.pack(pady=5)
    
    def find_text():
        text_area.tag_remove("found", "1.0", tk.END)
        word = find_entry.get()
        if word:
            start = "1.0"
            while True:
                start = text_area.search(word, start, stopindex=tk.END)
                if not start: break
                end = f"{start}+{len(word)}c"
                text_area.tag_add("found", start, end)
                start = end
            text_area.tag_config("found", background=ACCENT_RED, foreground="white")
    
    def replace_text():
        text = text_area.get("1.0", tk.END)
        new_text = text.replace(find_entry.get(), replace_entry.get())
        text_area.delete("1.0", tk.END)
        text_area.insert("1.0", new_text)
        win.destroy()
    
    create_mechanical_button(win, "НАЙТИ", find_text).pack(pady=5)
    create_mechanical_button(win, "ЗАМЕНИТЬ ВСЁ", replace_text).pack(pady=5)
